# NGAIP-364 - Implement Citation Accuracy Metric

## Ticket Purpose

NGAIP-364 implements citation accuracy and hallucinated-source detection for RAG answers. It verifies that assistant citations point to the correct PrattWise sources and that unsupported citations are reported.

## Runbook Decision

Use RAGAS faithfulness/grounding for semantic support, but keep citation precision, citation recall, and hallucination rate deterministic because RAGAS does not know PrattWise citation metadata.

RAGAS can judge whether an answer is supported by retrieved text. It cannot, by itself, verify whether `ChatResponse.sources`, `asset_id`, citation indexes, pages, spans, or source types match PrattWise expectations.

## Primary Metrics

This ticket owns:

- `citation_precision`
- `citation_recall`
- `hallucination_rate`

It should also expose or consume RAGAS faithfulness when the harness runs semantic grounding, but final source-link accuracy remains metadata-aware and deterministic.

## Expected Files

This ticket should update the citation metric module generated by the harness, typically under:

- `app_retrieval/evaluation/metrics/citation_accuracy.py`
- `tests/app_retrieval/test_citation_accuracy.py`

It should read source fields from:

- gold rows from `NGAIP-362`
- retrieved source metadata from the `NGAIP-363` harness
- report/schema expectations from `NGAIP-415`
- assistant citation structures such as `ChatResponse.sources`

## Metadata Rules

Citation scoring should preserve and compare:

- `asset_id`
- chunk id or span if available
- citation index if available
- source type
- page or content-item location if available

The initial hallucination rule is:

- A citation is hallucinated if its cited `asset_id` is not present in retrieved or gold-supported sources for that question.

`NGAIP-415` can tighten this rule after calibration, for example requiring page/span overlap as well as source identity.

## Non-Text Source Handling

For PDFs, OCR, tables, and mixed content, citation accuracy should evaluate the source metadata and the extracted text/markdown representation used by the assistant. A table citation can be correct if it points to the right asset and table-derived chunk/span, even though RAGAS is only judging the textual representation.

Preserve `content_type` so table/OCR citation failures can be reported separately.

## Validation

Run CI-safe tests without model calls:

```cmd
uv run pytest tests/app_retrieval/test_citation_accuracy.py -v
```

Tests should cover:

- exact source match
- wrong source
- missing citation
- hallucinated citation
- optional span/chunk overlap
- table/OCR metadata preservation where fixtures are available

The transfer script still supports local repeatable use by bootstrapping from local `main-backup-for-mac-claude-repo-04-07-2026`, switching or creating `ngaip-364-citation-accuracy-metric`, applying files, and committing locally without pushing.
